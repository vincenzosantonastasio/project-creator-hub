# Contributing to Project Creator Hub

Thank you for your interest in contributing to Project Creator Hub.

Project Creator Hub is an open-source initiative designed to organize the creator economy and improve how fans follow creators across the internet.

The project welcomes developers, designers and contributors who want to help build a better way to organize creator content online.

---

## Project Leadership

Project Creator Hub was created by:

**Vincenzo Santonastasio**  
Founder

The founder defines the overall product vision and long-term direction of the platform.

Community members are welcome to contribute ideas and help build the platform.

All contributions are reviewed before being accepted.

---

## Core Design Principle

Every feature proposed for the platform must pass a simple test.

Two questions must always be asked.

**Does this feature help fans?**

- Yes → continue  
- No → discard  

**Does this feature help creators?**

- Yes → continue  
- No → discard  

If a feature does not clearly help fans or creators, it should not be added to the platform.

This principle keeps the platform focused and prevents unnecessary complexity.

---

## Project Philosophy

Project Creator Hub is based on a simple idea:

> **People follow creators, not platforms.**

Creators publish content across many platforms such as:

- YouTube  
- TikTok  
- Instagram  
- Podcasts  
- Blogs  
- Newsletters  
- LinkedIn  

For fans this creates fragmentation and makes it difficult to follow everything a creator publishes.

The goal of the platform is to organize the digital presence of creators into a structured and accessible experience.

---

## Project Scope

Project Creator Hub is not intended to become:

- a social network  
- a content hosting platform  
- a messaging platform  
- a commenting platform  
- a creator monetization platform  

The platform does not replace existing platforms.

Its role is to organize creator content and help fans discover and follow creators more effectively.

Features that move the project away from this purpose may not be accepted.

## Contribution Workflow

Typical contribution process:

1. Fork the repository  
2. Create a new branch  
3. Implement improvements  
4. Submit a pull request  

Pull requests will be reviewed before being merged.

Only contributions aligned with the platform vision may be accepted.

---

## Code Standards

All contributions should aim for:

- clean and readable code  
- clear architecture  
- maintainable structure  
- documentation when necessary  

The project prioritizes **clarity, simplicity and scalability**.

---

## Contributor Credits

Project Creator Hub recognizes contributors who help build meaningful parts of the platform.

Developers and contributors whose work becomes part of the platform may be included in the **Project Credits**.

Credits may be given for:

- core platform features  
- major improvements  
- technical architecture contributions  
- significant development work  

Not every contribution will automatically appear in the credits.

The goal is to recognize contributors whose work meaningfully shaped the project.

---

## Where Credits Will Appear

Approved contributors may appear in the project credits in the following places:

- Credits section inside the platform  
- Credits section in the GitHub repository  
- Credits section in the project documentation  

This ensures contributors are publicly recognized for their work.

---

## What Is Not Allowed

To keep the platform focused and consistent with its vision, some types of contributions may not be accepted.

The project does not accept contributions that attempt to transform the platform into:

- a social network  
- a messaging system  
- a commenting system  
- a content hosting platform  

Project Creator Hub is designed to organize creator content, not replace the platforms where creators publish.

Features that move the platform away from this purpose may be rejected.

---

## Feature Proposals

Before implementing large features, contributors are encouraged to open a discussion.

This helps ensure that new ideas align with the platform vision before development begins.

---

## Final Decision Authority

Project Creator Hub is a community-driven project, but the final direction of the product is defined by the founder.

Major architectural and product decisions are reviewed to ensure the platform remains aligned with its original vision.

This prevents fragmentation and keeps the project coherent as it grows.

---

## Early Development Stage

The structure of the project may evolve as the platform design becomes clearer.
